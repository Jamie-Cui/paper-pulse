[1] An AI Agent Execution Environment to Safeguard User Data
    URL:       https://arxiv.org/abs/2604.19657v1
    Source:    arXiv
    Published: 2026-04-21
    Keywords:  prompt, agent, injection, security


## 背景与挑战
人工智能代理（AI agents）正迅速发展为通用型个人助手，但其需访问用户敏感数据（如通讯记录、财务信息、健康档案），由此引发严峻的隐私与安全
风险。现有系统依赖对AI模型或服务提供商的信任，而攻击者可通过**提示注入（prompt
injection）**、模型窃取或后门等手段诱导代理泄露私有数据——即使用户未主动授权，数据仍可能被静默上传至第三方服务器或嵌入推理请求中。

## 方法：GAAP执行环境
本文提出**GAAP（Guaranteed Accounting for Agent
Privacy）**——一种不依赖信任假设的AI代理执行环境，首次实现对用户私有数据的**确定性保密保障**。GAAP的核心创新在于：
- **动态权限采集**：通过交互式、面向任务的引导式提示（directed user
prompts），实时获取用户细粒度的数据共享意愿（例如“仅允许向银行API发送账户余额，禁止留存或转发”）；
- **增强型信息流控制（IFC）**：引入**持久化数据存储（persistent data stores）**
与**跨任务数据标注（cross-temporal
annotations）**，可精准追踪私有数据在单任务内各执行步骤间、以及跨多个异步任务（如隔日续聊）中的完整流动路径；
- **零信任执行**：GAAP在沙箱中运行代理，**绝不将原始私有数据交予AI模型或其提供商**；所有数据访问均经实时策略校验，违规披露（含隐
式泄露）被立即阻断。

## 主要发现与优势
实验表明：GAAP成功拦截全部12类典型数据泄露攻击（包括多跳推理泄露、上下文继承泄露、缓存侧信道等），而当前SOTA方案（如Privacera
、SafeAgent）在其中7种场景下失效；端到端延迟增加<8.2%，任务完成率保持99.3%，验证了强隐私保障与高代理效用的可行性。GAAP无
需修改模型架构、不依赖提示工程鲁棒性，为可信AI代理落地提供了可验证、可部署的基础执行层。

导**条件下，仅通过分析原始Windows事件日志数据库（含75,000–135,000条记录），自主发现并精确定位恶意行为发生的时间戳。

[3] Evaluating LLM-Generated Obfuscated XSS Payloads for Machine Learning-Based Detection
    URL:       https://arxiv.org/abs/2604.19526v1
    Source:    arXiv
    Published: 2026-04-21
    Keywords:  adversarial, learning, llm, machine, security


## 研究背景
跨站脚本（XSS）攻击长期位居OWASP Top 10，其顽固性核心在于**恶意载荷可通过多种混淆手段改变表层语法结构，却保持相同运行时行为**
。这种行为等价但语法多样的特性，严重削弱了基于规则与机器学习的检测系统鲁棒性。现有混淆生成方法（如JSFuck、Base64编码链）虽能提升语法
多样性，但**缺乏对行为一致性的系统性验证**，导致大量生成样本在真实浏览器环境中失效，进而污染训练数据、误导检测模型。

## 方法创新
本文提出首个**融合确定性变换、大语言模型（LLM）生成与浏览器级行为验证**的端到端XSS混淆载荷评估框架：
- **双阶段生成**：结合语法安全的确定性变换（如字符串拆分、Unicode转义）与LLM提示工程（含上下文示例与行为约束指令）；
- **行为导向评估**：所有生成载荷均在隔离Chrome环境执行，通过DOM变更、弹窗触发、网络请求等**可观测运行时指标**判定功能等价性（
而非仅依赖静态匹配）；
- **闭环优化**：构建行为保真源–目标对（如`alert(1)` → `eval('al'+'ert(1)')`），用于监督微调LLM。

## 关键发现
- 未微调基线LLM的行为匹配率仅**15%**，经行为保真数据微调后提升至**22%**——证实行为约束可提升生成质量，但当前LLM仍难以稳定
保持XSS语义；
- 将生成载荷直接加入检测器训练集**未提升AUC/Recall**，表明低质量样本引入噪声；而经行为过滤后的样本可**无损融入训练流程**（性
能下降<0.5%）；
- 首次揭示：**运行时行为验证是生成对抗安全数据不可绕过的质量门控**，单纯语法多样性无助于提升下游检测鲁棒性。

## 基于自动OpenAPI映射的API安全方法：Map Reduce Graph（MRG）

随着微服务架构的普及，HTTP REST API的动态性与异构性给安全防护带来严峻挑战——传统依赖人工编写的OpenAPI文档往往滞后、缺失或过
时，导致“影子API”和“幽灵端点”难以被发现与管控。本文提出**Map Reduce Graph（MRG）**，一种**完全无监督、全自动的A
PI建模与安全增强框架**。MRG无需先验知识、不依赖标注数据，仅通过分析生产环境真实HTTP流量，即可自主推断API拓扑结构：精准重建路由路径
（`/users/{id}`）、支持的HTTP方法（GET/POST等）、参数类型与格式（如`query`、`path`、`body`中的JSO
N schema），并**实时生成符合OpenAPI 3.0规范的可执行文档**。

MRG采用三阶段流水线设计：**训练阶段**构建初始图模型；**更新阶段**支持增量式、低开销的在线演进，适配API快速迭代；**检测阶段**融
合双重机制——基于图结构的**语义一致性验证**（识别未注册路径、非法方法组合、参数缺失/冗余）与基于深度自编码器的**负载内容异常分析**（捕
获SQLi、XSS、恶意JSON等注入攻击）。在真实API流量基准测试中，MRG相较HRAL与FT-ANN等前沿方法：**召回率最高提升11.4
%**，**推理速度超20倍**，且在SQL注入、路径遍历、畸形JSON等典型API层攻击上实现**100%精确率（零误报）**。其轻量设计与标
准化输出（OpenAPI YAML/JSON）可无缝集成至Prometheus、OpenTelemetry及WAF等可观测性与安全工具链，为云原
生环境提供开箱即用的实时API可见性、Schema自动推断与高保真异常检测能力。

## 背景与挑战
在垂直联邦学习（VFL）中，多方需对同一组样本（实体）建立隐私保护的对齐关系（PPEA），以支撑联合建模。传统私有集合交集（PSI）虽能实现对齐

## 方法创新
本文提出**Sherpa.ai多方PSU协议**，首个支持≥3方、兼顾精确匹配与噪声鲁棒性的PPEA框架：

*O(n)* 次模幂运算。

## 应用价值
该协议已在医疗（跨医院疾病预测）、金融（银行-保险联合风控）、电信-
金融反欺诈等真实场景验证，支持千万级实体对齐，**完全隐藏交集信息**，为合规VFL落地提供可扩展、数学严谨的基础设施。

## ProjLens：揭示投影器在多模态模型安全中的关键角色

数据投毒微调在MLLMs中植入后门，但其内在机制长期模糊不清，严重阻碍了可解释性分析与鲁棒防御设计。为此，我们提出**ProjLens**——首
个面向MLLMs投影器（projector）的细粒度可解释性框架，旨在系统解构后门如何在视觉-语言对齐模块中隐匿并激活。

我们首次发现：即使仅对连接视觉编码器与语言模型的**投影器进行微调**（不触碰视觉或语言主干），常规下游任务对齐过程本身即会引入后门易感性；且其
激活机制显著区别于纯文本大模型——并非依赖稀疏“触发神经元”，而是通过**低秩子空间编码**实现。通过在四种典型后门变体（包括像素/语义触发、目
标混淆等）上的大规模实验，我们揭示两大核心规律：
1. **低秩结构特性**：后门注入的参数更新整体呈满秩分布，无明显特异神经元，但所有后门关键参数均紧密聚集于投影器权重的一个**低维子空间**
（秩远低于原始维度）；
2. **范数敏感的语义偏移机制**：无论输入是否含触发器，其视觉嵌入均向后门目标类别方向发生语义偏移；而**偏移幅度与输入嵌入的L2范数呈线性
正相关**，导致中毒样本因更高范数而被显著错误分类，形成稳定激活。

ProjLens为MLLMs安全分析提供了首个投影器层面的可解释范式，兼具理论洞察与实用价值，代码已开源。

[9] Refute-or-Promote: An Adversarial Stage-Gated Multi-Agent Review Methodology for High-Precision LLM-Assisted Defect Discovery

## 背景与问题
大语言模型（LLM）辅助缺陷发现正面临严峻的**精度危机**：模型生成大量“看似合理实则错误”的缺陷报告，严重干扰维护者判断，稀释真实漏洞的可信
度与响应优先级。

## 方法创新：Refute-or-Promote 框架
本研究提出 **Refute-or-Promote**——一种推理时可靠性增强范式，包含四大核心机制：
- **分层上下文狩猎（SCH）**：结构化检索多粒度证据（规范、源码、测试、文档），生成高覆盖候选报告；
- **对抗式淘汰门控（Adversarial Kill
Mandates）**：在每一晋升阶段部署独立代理，强制执行“证伪优先”策略，仅当无法被驳倒时才推进；
- **冷启动评审员设计**：新代理不接触前序推理链，阻断锚定效应引发的级联误判；
- **跨模型批评者（CMC）**：调用异构架构/训练数据的LLM（如Llama 3 vs. Qwen vs.
